CSS Specificity

Whats is specificity?

Specificity is the means by which browsers decide which CSS property values are the most relevant to an element and, therefore, will be applied. Specificity is based on the matching rules which are composed of different sorts of CSS selectors.

Selectors List - Lowest to Highest

  • (*) Universal selector
  • (p, div, h1, etc.) Element selector
  • (.) Class selector
  • (#) ID selector
  • (style="color: yellow") Inline style
  • (!important) !important

(*) Universal selector

The * selector selects all elements. 


							* {
						    color: yellow;
							}

Element selector

The element selector selects all elements with the specified element name

(div) Element selector


							div {
						    color: yellow;
							}

Select and style all <div> elements

(div, p) Element selector


							div, p {
							  color: yellow;
							}

Select and style all <div> and <p> elements

(.) Class selector

The class selector selects elements with a specific class attribute

(.) Class selector


							.some-class {
							  color: yellow;
							}

Select and style all elements with class="some-class"


							<div class="some-class"> A yellow div </div>
							<p class="some-class"> A yellow paragraph </p>

(p.some-class) Class selector


							p.some-class {
							  color: yellow;
							}

Style all <p> elements with class="some-class" 


							<p class="some-class"> A yellow paragraph </p>

(#) ID selector

The id selector styles the element with the specified id 


							#some-id {
							  color: yellow;
							}

Style the element with id="some-id" 


							<p id="some-id"> Some yellow text </p>

Inline style

Styling within HTML will 'win' over anything in CSS 


							<p style="color: yellow"> Some yellow text </p>

!important

When an important rule is used it overrides ANY other declaration, wether in HTML, CSS 


							.some-class {
							  background-color: yellow !important;
							}
							.some-class {
							  background-color: red;
							}
